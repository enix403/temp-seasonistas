SCOPE: General

GET => /api/me/profile
Get the current user's profile

PUT => /api/me/profile
Update the current user's profile

GET => /api/user/:userId
Get any user's profile

----------
SCOPE: Employer

POST => /api/job/post
Allows an employer to post a new job

GET => /api/employer/my-postings
Returns a list of jobs posted by the employer

----------
SCOPE: Employee

GET => /api/job/search
Allows the employee to search for jobs (using filters like job title, location, job type, etc.)

GET => /api/job/:jobId
Returns detailed information about a specific job posting.

POST => /api/job/:jobId/apply
Allows the employee to apply for a job.

GET => /api/job/application/:applicationId
View details of a specific application.

PUT => /api/job/application/:applicationId/update-status
Allow the employer to update the status of the application

GET => /api/employee/my-applications
Fetch a list of applications created by the employee.

----------
SCOPE: Employer

GET => /api/job/:jobId/applicants
Fetch a list of employees who have applied for a specific job posting.

PUT => /api/job/:jobId/update-status
Allow the employer to mark a job posting as active or inactive.

----------

GET => /api/admin/users
Fetches a list of all users (employees and employers) for administrative purposes.


=============

PUT => /api/admin/user/:userId/ban
Allows the admin to ban a user from the platform.

=============

GET => /api/company/:companyId/employees
Fetches a list of employees who currently work or have worked for a specific employer.

GET => /api/job/post-template
Allows employers to use a previous job posting as a template for a new one (if available).

GET => /api/employer/:employerId/employees
Fetches a list of employees currently or previously working for the employer.

PUT => /api/admin/user/:userId/role
Allows the admin to update the role of a user (e.g., promoting an employee to an employer).
